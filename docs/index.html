Everything in one stack

No per-tool configuration. No multiple API keys to manage in your IDE.

🌐

Offline-first

Local models run via Ollama with Apple Metal GPU acceleration. Keep working when your internet is down — no cloud dependency for day-to-day coding.

🔀

Automatic fallback

If a local model fails or is unavailable, LiteLLM silently escalates to the next provider: Anthropic → OpenAI → Gemini. Fully transparent to your IDE.

🔌

One endpoint for everything

localhost:4000 works with Cline, curl, Python scripts, or any OpenAI-compatible client. Switch models without reconfiguring your tools.

📊

Built-in spend tracking

LiteLLM's web UI at localhost:4000/ui shows token usage, spend by provider, request logs, and virtual API keys with spending limits.

🧙

Interactive setup wizard

Run bash setup.sh — it detects your RAM, shows which models fit your hardware, walks you through provider selection and generates your .env automatically.

🔍

Pre-flight checker

Run bash check.sh before setup to verify RAM, disk space, Docker, Ollama, port availability, and model compatibility — all in one report.

How it works

One proxy, multiple backends. Your IDE talks to LiteLLM; LiteLLM routes to wherever makes sense.

Quick start

From zero to a running local AI stack in under 5 minutes.

1

Clone the repository

git clone https://github.com/k-trimbuddhimatta/localai.git
cd localai
2

Check your system (recommended)

Verifies hardware compatibility and shows which local models fit your RAM.

bash check.sh
3

Run the setup wizard

Walks you through provider selection, API keys, model download and starts the Docker stack.

bash setup.sh

# The wizard will ask:
#   — Which cloud providers to configure (Anthropic / OpenAI / Google / none)
#   — API keys for each selected provider
#   — Which local model to download (filtered to your RAM)
#   Generates .env and starts Docker automatically.
4

Configure Cline in VS Code

Open Cline → gear icon ⚙️ → API Configuration and fill in:

FieldValue
API ProviderOpenAI Compatible
Base URLhttp://localhost:4000
API Keyyour LITELLM_MASTER_KEY (shown at end of setup.sh)
Modelclaude-sonnet-4-6 (recommended for agent mode)

Available models

All models are available through a single endpoint. Switch by changing the model name in Cline.

Local — Ollama

Runs on your Mac · Free · Works offline · Apple Metal GPU

qwen2.5-coder:14b Recommended 16 GB+ RAM · ~9 GB
qwen2.5-coder:7b Lightweight 8 GB+ RAM · ~5 GB
llama3.1:8b General 8 GB+ RAM · ~5 GB

Cloud — via LiteLLM

Requires API key · Best for agent mode · Automatic fallback target

claude-sonnet-4-6 Agent mode Anthropic
claude-opus-4-8 Most capable Anthropic
gpt-4o General OpenAI
gpt-4o-mini Fast OpenAI
gemini-2.5-flash Multimodal Google
gemini-2.5-pro Pro Google

Additional Ollama models can be pulled with ollama pull <model> and registered in litellm/config.yaml.

Requirements

Designed for macOS with Apple Silicon. Intel Macs work but models run on CPU (slower).

Operating system

macOS 13+

Apple Silicon (M1–M5) recommended

Docker Desktop

Version 4.x+

Must be running before setup.sh

Ollama

Any recent version

Install from ollama.com

RAM (minimum)

8 GB

16 GB+ recommended for 14b models

RAM requirements per local model
Model Size on disk Min RAM Notes
qwen2.5-coder:7b ~5 GB 8 GB Works on 16 GB Macs; leave other apps closed
qwen2.5-coder:14b ~9 GB 16 GB Recommended — best quality for 24+ GB systems
qwen2.5-coder:32b ~20 GB 32 GB High-spec machines only
llama3.1:8b ~5 GB 8 GB General purpose, not code-specialised
llama3.1:70b ~40 GB 64 GB Not viable on consumer hardware

On Apple Silicon, unified memory means all RAM is available for models (no separate VRAM limit). If a model doesn't fit, Ollama falls back to swap — 10–50× slower and effectively unusable. Run bash check.sh to see exactly which models are compatible with your machine.
